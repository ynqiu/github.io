打开VNC时遇到”too many security failures”

有时很久没有VNC，打开时会提示错误的指令”too many security failures”而无法打开。这种情况一般而言，我们可以直接kill掉这个VNC，但是如果kill掉这个VNC，那么这个虚拟桌面未保存的工作也将会失去。

经过网上的搜索，发现这是因为VNC将”0.0.0.0”加入和黑名单的名列，所以此时只要禁止VNC使用黑名单即可。

这里的”5”是指我的VNC端口是5。如果你的是其他端口，可以设置成对应的端口号。

然后重新连接VNC就可以啦！你会发现上次未保存的工作仍然还在的！
